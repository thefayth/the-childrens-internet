# Public / Private Boundary

## Public

- Project name and purpose
- Current status
- Public microsite URL
- High-level workflow diagrams
- Public-safe visuals
- WordPress draft copy
- Ownership and commercial-use terms

## Private

- Source code
- Original DOCX concept source
- Prompts and agent instructions
- Credentials and secrets
- Deployment scripts and server details
- Private review receipts
- Legal, administrative, family, medical, benefits, customer, or private strategy materials
- Any future backend system for children, families, moderation, verification, reporting, or compliance

## Rule

The public surface may explain the idea and show the intended experience. It must not expose the engine.
